# Система мониторинга академических рисков студентов

Интерактивная платформа для **сбора, обработки, анализа и визуализации** данных о студентах (успеваемость, анкеты, психометрика, опросники) с применением **классических и ансамблевых моделей машинного обучения**, **объяснимости (SHAP)** и **мониторинга дрейфа данных**.

Основной пользовательский интерфейс в репозитории — **`app.py` (Streamlit)**. Отдельно могут существовать API/React (если добавлены в проект) — см. раздел «Связанные компоненты».

## Возможности

- **Загрузка данных**
  - синтетические наборы по **категориям** (успеваемость, психология, креативность, ценности Шварца, личность, активность, карьера);
  - **CSV** с признаками и целевой переменной;
  - **Excel** с листами опросов через `ml_core.loader`.
- **Предобработка и признаки**
  - композитные признаки (`trend_grades`, `cognitive_load`, `overall_satisfaction` и др.);
  - отбор признаков, **SMOTE**, train/test split.
- **Аналитика**
  - корреляционная матрица (Plotly heatmap);
  - **K-means** кластеризация, профили кластеров, **PCA**-визуализация;
  - кросс-валидация моделей (**LR, RF, XGB**);
  - опционально **RandomizedSearchCV** для XGBoost;
  - ROC, Confusion Matrix, важность признаков;
  - **SHAP**-объяснения для отдельных наблюдений.
- **Расширенный анализ (вкладки в Streamlit)**
  - кросс-таблицы + χ² (`ml_core.crosstab`);
  - временные ряды / траектории студентов (`ml_core.timeseries`);
  - обработка пропусков и выбросы (`ml_core.imputation`);
  - сохранение/загрузка экспериментов (`ml_core.experiment_tracker`).
- **Мониторинг**
  - логирование событий и метрик (`monitoring.logger`);
  - детектор **дрейфа** распределений (KS для числовых, χ² для категориальных) (`monitoring.drift_detector`).
- **LLM (опционально)**
  - модуль `ml_core.llm_interface` (YandexGPT и задел под GigaChat); в `app.py` блок с LLM **закомментирован** — включается при наличии ключей и доработке UI.

## Пакет `ml_core`

Ниже — назначение **каждого модуля** в текущей версии проекта.

### `ml_core/data.py`

- **`load_data(category, n_students, generate_two_sets)`** — основная точка загрузки **синтетики** (кэшируется Streamlit `@st.cache_data`). Категории: `grades`, `psychology`, `creativity`, `values`, `personality`, `activities`, `career`. При `generate_two_sets=True` может вернуть эталон и «новые» данные для демонстрации дрейфа.
- Внутри — генерация признаков под ТЗ (успеваемость, анкеты, психотесты, эссе и др.) и агрегация по студентам.
- **`load_from_csv`**, **`prepare_data_for_training`** — загрузка пользовательского CSV и стратифицированный split.
- **`save_synthetic_data`**, **`save_data_for_monitoring`** — сохранение в `analysis_data/`.

**Зависимость:** модуль импортирует `streamlit` (для кэша) — при вызове функций **вне Streamlit** учитывайте это (или вынесите `load_data` без декоратора в отдельный слой).

### `ml_core/features.py`

- **`add_composite_features(df)`** — производные признаки: тренд/стабильность оценок, когнитивная нагрузка, удовлетворённость, психологическое благополучие, академическая активность.
- **`get_base_features(df)`** — автоматический список базовых колонок по соглашению имён (успеваемость, анкеты, тесты, эссе).
- **`preprocess_data`** — медианное заполнение + **SMOTE**.
- **`select_features`** / **`select_features_for_model`** — отбор признаков (mutual information + RFE или комбинация F-score + RF в зависимости от функции).

### `ml_core/models.py` (`ModelTrainer`)

- Набор моделей: **Logistic Regression**, **Random Forest**, **XGBoost**.
- **`cross_validate`**, **`train_best_model`** — кросс-валидация по F1, обучение лучшей модели, метрики на тесте.
- **`tune_xgboost`** — RandomizedSearchCV по гиперпараметрам XGBoost.
- **`save_model` / `load_model`** — `joblib` + JSON метаданные в каталоге `models/` (по умолчанию).

### `ml_core/analysis.py`

- **`correlation_analysis`** — корреляция по числовым признакам; опционально сохраняет CSV и heatmap PNG (через seaborn/matplotlib).
- **`cluster_students`**, **`analyze_cluster_profiles`**, **`plot_clusters_pca`**, **`plot_clusters_2d`** — кластеризация K-means, профили, визуализация через PCA и Plotly/Matplotlib.

### `ml_core/evaluation.py`

- Метрики: **F1, Precision, Recall, ROC-AUC**.
- Plotly-фигуры: ROC, confusion matrix, feature importance.
- **`generate_shap_explanations`** — SHAP (TreeExplainer / LinearExplainer) и структурированные объяснения для топ-наблюдений.
- Вспомогательные функции для текстовых и детальных объяснений (для совместимости с разными версиями SHAP).

### `ml_core/loader.py`

- Разбор **Excel** с несколькими листами: **`detect_sheet_type`** по имени листа (Вильямс, Шварц, Соц1–15 и т.д.).
- **`preprocess_sheet` / `preprocess_excel_data`** — очистка, поиск колонки пользователя (`user`, `user_id`, `VK_id`), числовое приведение для опросных шкал.
- Используется в `app.py` при выборе «Excel файл (опросы)».

### `ml_core/imputation.py`

- **`handle_missing_values`** — стратегии: `auto`, заполнение средним/медианой/модой, интерполяция, удаление строк/столбцов; возвращает отчёт действий.
- **`detect_outliers`** (используется во вкладке пропусков в `app.py`).

### `ml_core/crosstab.py`

- **`create_crosstab`** — сводные таблицы для социологического анализа, χ² при подсчёте частот, heatmap и stacked bar (Plotly).

### `ml_core/timeseries.py`

- **`analyze_student_trajectory`** — тренд по времени (полином 1-й степени), статус improving/declining/stable, график Plotly.
- **`detect_negative_dynamics`** — поиск студентов с ухудшающейся динамикой по выбранному показателю и семестрам.

### `ml_core/experiment_tracker.py`

- **`ExperimentTracker`** — каталог `experiments/<id>/`: `metadata.json`, опционально `model.pkl`, `explanations.csv`, `predictions.csv`.
- Список и загрузка прошлых экспериментов из UI.

### `ml_core/llm_interface.py`

- **`LLMInterface`** — вызов **YandexGPT** (по `LLM_API_KEY`, `YANDEX_FOLDER_ID`), задел под GigaChat.
- Методы: интерпретация кластеров, отчёты, ответы на вопросы по сводке данных.
- В **`app.py` сейчас закомментирован** — подключение требует секретов и политики безопасности.

## Папка `monitoring`

### `monitoring/logger.py` — `MLLogger`

- Пишет события в **`logs/ml_events.log`**.
- Накапливает метрики моделей в **`logs/model_metrics.csv`** (timestamp, model_name, f1, roc_auc, precision, recall, data_size, features).
- Методы: `log_event`, `log_model_metrics`, `log_features`, `log_prediction`.

### `monitoring/drift_detector.py`

- **`DataDriftDetector`**
  - эталон: `reference_data` + опционально `model_metadata` (списки числовых/категориальных признаков);
  - **числовые**: тест **Колмогорова–Смирнова** (p-value &lt; threshold ⇒ дрейф);
  - **категориальные**: **χ²** по таблице сопряжённости;
  - отчёт: `overall_drift`, `drift_percentage`, список признаков, рекомендации, качество данных (пропуски, выбросы, новые категории);
  - **`save_report`** → JSON в `logs/drift_reports/`.
- **`DriftMonitorThread`** — заготовка фонового цикла (sleep по часам; автозагрузка данных — TODO).
- **`DriftMonitorScheduler`** — регистрация нескольких детекторов и массовая проверка.

В **`app.py`** после успешного обучения создаётся `DataDriftDetector` на выбранных признаках (`selected_cols`); пользователь может загрузить новый CSV и получить отчёт о дрейфе.

## Файл `app.py` — логика интерфейса и пайплайна

### Инициализация

- **`st.set_page_config`**, заголовок.
- **`MLLogger()`** — глобальный логгер.
- **`ModelTrainer(models_dir='models')`**.
- **`st.session_state`** — хранение `df`, графиков, моделей, метрик, детектора дрейфа, эталонных данных и т.д.

### Сайдбар

- **Тип данных:** синтетика / CSV / Excel.
- Для синтетики: **категория**, число студентов, опция **двух наборов** для дрейфа.
- **Параметры анализа:** число кластеров, порог риска, тюнинг XGBoost.
- **Доп. настройки:** метрика оптимизации (f1 / roc_auc / precision / recall), число признаков в финальной модели, чекбоксы моделей, `shap_top_n`.
- Кнопка **«Запустить анализ»**.

### При нажатии «Запустить анализ» (`run_analysis`)

1. Лог: `analysis_started`.
2. Загрузка данных (ветка по типу).
3. **`get_base_features`** → **`add_composite_features`** → список `all_features`.
4. Целевая переменная: `risk_flag` или `target`; при отсутствии — **эвристическое создание** в зависимости от `category` (успеваемость, стресс/мотивация, креативность, ценности и т.д.).
5. Дедупликация колонок.
6. **Корреляция** по числовым колонкам (+ target).
7. **Кластеризация** на `all_features` (с запасным набором признаков).
8. **Обучение:** SMOTE → split → **`select_features_for_model`** → CV → **`train_best_model`** → опционально тюнинг XGB → переобучение выбранных моделей для ROC → сохранение модели → лог метрик.
9. Инициализация **`DataDriftDetector`** на `selected_cols` (после первого успешного прогона).
10. **SHAP** через `generate_shap_explanations`.
11. Сохранение всего в `session_state` (`analysis_completed = True`).

### Отображение после анализа

- Метрики по данным, корреляция, кластеры, CV, тестовые метрики, ROC, CM, feature importance.
- Вкладки: **кросс-таблицы**, **эксперименты**, **временные ряды**, **пропуски**.
- Блок **SHAP** (первые объяснения в expander).
- **Экспорт:** CSV предсказаний и объяснений.

### Мониторинг (внизу страницы)

- Вкладки: дрейф, история метрик из `logs/model_metrics.csv`, настройка порога p-value для дрейфа.

## Установка и запуск

### Требования

- Python **3.10+** (рекомендуется 3.11).
- Зависимости из **`requirements.txt`** (и при необходимости `pip install python-dotenv`, если не указан).

### Основной сценарий (Streamlit)

```bash
python -m venv .venv
.venv\Scripts\activate   # Windows
pip install -r requirements.txt
streamlit run app.py
```

Откройте URL, который выведет Streamlit (обычно `http://localhost:8501`).

### Опционально: конфигурация

Создайте файл **`.env`** в корне (см. [Переменные окружения](#переменные-окружения)).  
`config/settings.py` подхватит его при импорте (если вы используете этот модуль в коде).

## Данные: форматы и ожидания

### CSV

- Числовые признаки для моделей и корреляций.
- Целевая переменная: **`risk_flag`** (предпочтительно) или **`target`**.
- Для временных рядов во вкладке «Временные ряды» желательны **`student_id`** и **`semester`**.

### Excel (опросы)

- Листы с ключевыми словами в названии (см. `ml_core.loader.detect_sheet_type`).
- Колонка респондента: `user`, `user_id` или `VK_id` (иначе создаётся искусственный `user_id`).

### Синтетика

- Задаётся категорией в сайдбаре; целевая переменная и набор признаков зависят от генератора в `ml_core.data`.

## Связанные компоненты

В одном репозитории могут сосуществовать:

- **`app.py`** — полнофункциональный прототип для аналитика/исследователя.
- **`api/`**, **`workers/`**, **`frontend/`**, **`docker-compose.yml`** — если вы выносите ML за HTTP и очереди (ответственность может быть разделена с бэкенд-разработчиком).

Для работы React/UI с API нужен **запущенный backend** на том же хосте/порту, что указан в `VITE_API_BASE` (или по умолчанию `http://localhost:8001`).
