## Описание реализации по ТЗ

Этот файл описывает, **как текущая реализация платформы соответствует техническому заданию**, и какие **входные данные** используются на аналитическом этапе.

Фокус этой части системы – два слоя из общей архитектуры:
- `Система представления данных для анализа`
- `Аналитическая система`

Все, что связано с интерфейсами анкетирования, ролями пользователей (Администратор, Специалист, Респондент) и долгоживущим хранилищем данных – **вне рамок данного модуля** и может быть реализовано отдельно.

---

## Входные данные (соответствие 4 типам из ТЗ)

Согласно ТЗ исходные данные включают:

- **1. Данные об успеваемости студентов**
  - Примеры полей:
    - `student_id` – идентификатор студента
    - `course_name` – название дисциплины
    - `grade` – оценка
    - `semester` – номер семестра
    - `brs_score` – балл по БРС

- **2. Данные по итогам анкетирования студентов**
  - После агрегации по студентам используются числовые показатели:
    - `satisfaction_score` – удовлетворённость
    - `engagement_score` – вовлечённость
    - `workload_perception` – субъективная нагрузка

- **3. Данные по итогам психологического тестирования**
  - После агрегации по студентам:
    - `stress_level` – уровень стресса
    - `motivation_score` – мотивация
    - `anxiety_score` – тревожность

- **4. Эссе как учебная работа**
  - На текущем этапе используются агрегированные числовые признаки:
    - `n_essays` – количество эссе
    - `avg_essay_grade` – средний балл за эссе
  - Текстовый анализ (NLP) можно добавить на следующем этапе.

Дополнительно формируется **целевая переменная**:

- `risk_flag` – бинарный признак «академический риск» (0/1), который в синтетических данных рассчитывается как функция комбинации факторов (низкая успеваемость, высокий стресс, низкая удовлетворённость). В реальной системе он может определяться, например, по факту академической задолженности или отчисления.

Все эти данные **агрегируются в один датафрейм по студентам** в модуле `student_risk_pipeline.py` функцией `_aggregate_student_features`.

## 1. Система представления данных для анализа

Соответствует блоку:

> **Система представления данных для анализа**  
> – Интерактивная подсистема сводки и группировки данных по объектам  
> – Интерактивная подсистема оценки неизвестных значений  
> – Подсистема формирования выходных данных для системы анализа

### 1.1. Интерактивная подсистема сводки и группировки данных по объектам

**Что требует ТЗ:**
- Сводка и группировка данных по объектам анализа (студентам, группам и т.п.).

**Как реализовано:**
- Функция `_aggregate_student_features(...)` в `student_risk_pipeline.py`:
  - собирает данные из разных источников (успеваемость, анкеты, психологические тесты, эссе);
  - агрегирует их **по `student_id`**;
  - считает агрегированные показатели:
    - `avg_grade`, `grade_std`, `min_grade`, `max_grade`, `n_courses`, `avg_brs`;
    - добавляет `trend_grades` (изменение средней оценки между семестрами).
- В Streamlit-приложении (`app.py`) эти агрегированные данные:
  - отображаются в таблицах;
  - используются как основа для кластеризации студентов и построения профилей групп риска.

**Итог:** реализована подсистема сводки и группировки данных по студентам, соответствующая требованиям ТЗ.

### 1.2. Интерактивная подсистема оценки неизвестных значений

**Что требует ТЗ:**
- Оценка / заполнение неизвестных значений (пропусков в данных).

**Как реализовано:**
- В `_aggregate_student_features(...)`:
  - после объединения источников вызывается:
    - заполнение пропусков медианой по числовым признакам: `result.fillna(result.median(numeric_only=True))`.
- В `preprocess_data(...)`:
  - перед применением SMOTE пропуски в признаках также заполняются медианой:
    - `X = X.fillna(X.median(numeric_only=True))`.

**Итог:** реализована автоматическая оценка и заполнение неизвестных значений, как это указано в ТЗ (Data Cleaning).

### 1.3. Подсистема формирования выходных данных для системы анализа

**Что требует ТЗ:**
- Подготовка некого «слоя представления данных» для аналитической системы.

**Как реализовано:**
- Функции:
  - `preprocess_data(df, feature_cols, target_col)` – готовит `X_res`, `y_res` для моделей;
  - `select_features(X, y, ...)` – отбирает подмножество признаков для последующего анализа;
  - экспорт подготовленных данных и результатов в CSV:
    - `predictions.csv` – предсказания модели;
    - `cluster_profiles.csv` – профили кластеров;
    - `corr_matrix.csv` – корреляционная матрица;
    - `detailed_explanations.csv` / `explanations.csv` – объяснения для студентов группы риска.

**Итог:** выходные данные формируются в удобном для аналитической системы и пользователей виде (таблицы и файлы CSV).

## 2. Аналитическая система

Соответствует блоку:

> **Аналитическая система**  
> – Интерактивная подсистема выборки показателей и объектов группировки  
> – Интерактивная подсистема выбора и реализации метода анализа (корреляция, регрессия и т.п.)

### 2.1. Интерактивная подсистема выборки показателей и объектов группировки

**Что требует ТЗ:**
- Возможность выбирать, какие показатели анализировать;
- Возможность группировать респондентов по различным признакам.

**Как реализовано:**
- В `student_risk_pipeline.py`:
  - автоматическое формирование списка признаков `base_features` на основе наличия колонок в данных:
    - успеваемость (`avg_grade`, `grade_std`, ...),
    - анкеты (`satisfaction_score`, ...),
    - психологические тесты (`stress_level`, ...),
    - эссе (`n_essays`, `avg_essay_grade`),
    - + композитные признаки (`trend_grades`, `cognitive_load`, и др.).
- В `app.py` (Streamlit):
  - пользователь может:
    - запускать анализ на собственных данных (загружая CSV);
    - выбирать количество кластеров;
    - задавать порог риска для отнесения студента к группе риска.
- Группировка объектов:
  - функция `cluster_students(...)` выполняет **кластеризацию студентов** по выбранным признакам;
  - `analyze_cluster_profiles(...)` формирует **профили кластеров** – средние значения признаков в каждом кластере.

**Итог:** реализована возможность выборки показателей и группировки респондентов в кластеры с последующим анализом.

### 2.2. Интерактивная подсистема выбора и реализации метода анализа

**Что требует ТЗ:**
- Поддержка разных методов анализа: корреляция, регрессия, кластеризация и т.п.

**Как реализовано:**

- **Корреляционный анализ**:
  - функция `correlation_analysis(df, feature_cols, target_col)`:
    - считает корреляционную матрицу для признаков и целевой переменной `risk_flag`;
    - сохраняет матрицу в `corr_matrix.csv`;
    - при наличии `seaborn` строит тепловую карту `corr_heatmap.png`;
    - дублируется интерактивным графиком в Streamlit.

- **Регрессионный / классификационный анализ (прогноз риска)**:
  - логистическая регрессия (`LogisticRegression`) – базовый, интерпретируемый метод;
  - Random Forest (`RandomForestClassifier`) – ансамблевый метод, даёт баланс точности и интерпретируемости;
  - XGBoost (`XGBClassifier`) – градиентный бустинг, обеспечивает максимальное качество.
  - Метрики:
    - `ROC-AUC`, `F1`, `Precision`, `Recall` – учитывают дисбаланс классов, как описано в ТЗ.
  - Кросс-валидация:
    - `cross_validate_models(...)` с `StratifiedKFold` – устойчивые оценки качества.
  - Оптимизация гиперпараметров:
    - `tune_xgboost(...)` – `RandomizedSearchCV` по ключевым параметрам XGBoost.

- **Кластерный анализ**:
  - `cluster_students(...)` – K-means кластеризация студентов по набору признаков;
  - `plot_clusters_2d(...)` / визуализация в Streamlit – 2D-проекция (PCA) с раскраской по кластерам.

- **Интерпретируемость (объяснение прогнозов)**:
  - `explain_model_with_shap(...)` – краткие объяснения влияния признаков;
  - `generate_detailed_explanation(...)` и `generate_batch_explanations(...)`:
    - формируют текстовые объяснения для конкретных студентов:
      - вероятность риска,
      - топ-факторы, увеличивающие/снижающие риск,
      - простые рекомендации (связанные с трендом оценок, посещаемостью, стрессом и т.п.).

**Итог:** реализован набор классических и современных методов анализа (корреляция, регрессия/классификация, кластеризация, объяснимый ML), что полностью соответствует блоку «Интерактивная подсистема выбора и реализации метода анализа».

## 5. Краткая схема потока данных

1. **Сбор и хранение данных** – вне рамок, предполагается существование источников в формате CSV/БД.
2. **Загрузка и агрегация** (`load_data`, `_aggregate_student_features`):
   - объединение 4 типов данных по `student_id`;
   - формирование числовых признаков.
3. **Создание композитных признаков** (`add_composite_features`):
   - тренды, стабильность, когнитивная нагрузка, удовлетворённость, благополучие и т.п.
4. **Подготовка к анализу** (`preprocess_data`, `select_features`):
   - обработка пропусков;
   - борьба с дисбалансом (SMOTE);
   - отбор признаков.
5. **Аналитика**:
   - корреляционный анализ (`correlation_analysis`);
   - сравнение моделей (LR / RF / XGB), кросс-валидация, тюнинг XGBoost;
   - кластеризация студентов (`cluster_students`);
   - объяснимость прогнозов (SHAP).
6. **Представление результатов**:
   - CSV-отчёты;
   - графики (ROC, Confusion Matrix, кластеры, важность признаков, корреляции);
   - интерактивный Streamlit-интерфейс для Аналитика.
